# Contribuir al proyecto

Gracias por tu interés en contribuir. Estas pautas ayudan a que las contribuciones sean claras y fáciles de revisar.

## Cómo empezar
1. Haz un fork del repositorio y crea una rama con un nombre descriptivo: `feature/mi-cambio` o `fix/descripcion`.
2. Mantén tu rama actualizada con la rama principal (`main` o la rama por defecto del repo).

## Estilo de commits
- Mensajes claros y en inglés o español según el contexto del cambio.
- Formato sugerido: `tipo: descripción corta` (ej. `fix: corregir enlace roto en home`).

## Flujo de trabajo
- Abre un issue para discutir cambios complejos antes de empezar.
- Crea una rama por cambio (no trabajes directamente en la rama principal).
- Envía un pull request (PR) cuando tu cambio esté listo.

## Revisión y pruebas
- Incluye en el PR una descripción clara del cambio y cómo probarlo.
- Añade capturas de pantalla o enlaces si aplica.
- Responde a los comentarios de la revisión y actualiza el PR según sea necesario.

## Pruebas locales
- Instala dependencias y ejecuta el proyecto localmente según el README del repositorio.
- Incluye instrucciones para ejecutar tests si añades código que los requiera.

## Código y estilo
- Mantén la coherencia con las convenciones del proyecto (estructuras, naming y formateo).
- Evita cambios no relacionados dentro de un mismo PR.

## Agradecimientos
Valoramos todas las contribuciones — desde reportes de bugs hasta mejoras de documentación.
